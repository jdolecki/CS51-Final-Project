module BDD: sig .. end
Abstract type for binary decision diagrams.
type bdd
Abstract type for binary decision diagrams.
type expression
Abstract type for boolean expression .
type operation
Abstract type for boolean operators
val make : int -> bdd -> bdd -> bdd
Make builds a node out of an expression and ensures the bdd is reduced as outlined in 'An Introduction to Binary Decision Diagrams' by Andersen. It returns that node, as well.
val build : expression -> bdd
Build takes an expression and recursively constructs its binary decision diagram. It primarily uses Make for constructing each Node. Given expression should have variables given in order 1, 2, 3... if the user wants correct sat_count solutions.
val apply : operation -> bdd -> bdd -> bdd
Apply takes a boolean operator of type operation and two bdds, and returns a new bdd that is the result of the given operation.
val sat_count : bdd -> float
Sat_count returns the number of solutions to the Boolean function given f(x) = 1.
val any_sat : bdd -> (int * expression) list
Any_sat returns a list of variable assignments that result in f(x) = 1. The path any_sat takes is greedy, and it may fail in the case of functions that aren't well behaved. In those cases, NoSolutionFound exception is thrown and all_sat should be used to find the solution.
val all_sat : bdd -> (int * expression) list list
All_sat returns all satisfying variable assignments that result in f(x)=1 where f(x) is the boolean expression for the given bdd.
val restrict : bdd -> expression * expression -> bdd
restrict takes a variable assignment and a bdd and returns the new, simplified bdd. Variable assignments should be passed as (var number, True/False).
